Um Serviço de Stream de Dados

Introdução

Neste trabalho foi implementado um servidor de streams de dados referentes à aparecimentos de caravelas-portuguesas no litoral brasileiro, contendo dados da latitude, longitude e timestamp do aparecimento. A ideia deste trabalho é simular de forma simplificada serviços de streaming de dados, como Netflix e Spotify. Da mesma forma, o servidor implementado faz o stream da localização de caravelas, e os clientes podem se conectar ao servidor para receber os dados e realizar operações sobre eles.

Desenvolvimento

O servidor foi implementado em Python, e é responsável por receber os dados de um arquivo .pkl com as informações das caravelas-portuguesas, armazenados em um dataframe da biblioteca Pandas, e enviar para os clientes que se conectarem a ele. O servidor é implementado com os seguintes métodos:

  • logging: Escreve uma mensagem especifica em um arquivo de texto de nome "log-execucao-server-timestamp.txt", onde cada execucao adiciona o timestamp ao nome do log, facilitando em navegar entre as execuções.
  • check_arguments: Checa se os argumentos da execução do servidor foram passados corretamente na linha de comando e retorna os valores passados para realizar a configuração do socket. Espera-se receber o número da porta onde o socket vai operar e o intervalo de tempo entre cada envio de mensagem. Se houverem erros nesses parâmetros é apresentada uma mensagem de erro para o usuário mostrando como deve ser realizada a entrada dos dados.
  • start_server: Inicia o socket do server e retorna o socket, o endereco do socket e o intervalo de tempo entre cada mensagem.
  • wait_for_clients: Espera a conexao de um client.
  • stream: Envia a mensagem para todos os clientes da lista.

O cliente foi implementado também em Python, e é responsável por receber os dados do servidor, e realizar alguma operação sobre eles, no caso deste trabalho, armazenar os dados binários em um novo dataframe e executar a operação count(), que contabiliza a quantidade de valores por coluna do dataframe. O cliente é implementado com os seguintes métodos:

  • logging: Escreve uma mensagem especifica em um arquivo de log de nome "log-execucao-server-timestamp.txt", onde cada execucao adiciona o timestamp ao nome do log.
  • check_arguments: Checa se os argumentos foram passados corretamente e retorna os valores passados. Caso contrario, imprime uma mensagem de erro e encerra o programa.
  • socket_configuration: Configura o socket do client e retorna o socket e o endereco do socket.
  • start_conection: Inicia a conexao com o server e retorna o numero de pacotes a serem recebidos.

Ambos servidor e cliente foram desenvolvidos para transmitir apenas um stream de dados, referente a um único dataframe que é transmitido linha por linha pela rede, utilizando o indíce do dataframe como número de sequência para as mensagens.

Execução

Para executar o servidor, basta executar o seguinte comando: python3 server-udp.py [port] [delay]

Para executar o cliente, basta executar o seguinte comando: python3 client-udp.py [hostname] [port]
